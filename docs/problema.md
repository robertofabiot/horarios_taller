# Problema Propuesto

Una universidad desea desarrollar un sistema básico para gestionar cursos y docentes en un período académico, permitiendo registrar cursos con su información (nombre, código y horario) y docentes con sus datos principales.

El sistema debe permitir asignar docentes a cursos y consultar dicha información. Además, se busca modelar distintos tipos de cursos (por ejemplo, teóricos y prácticos) que puedan compartir características comunes, pero también comportamientos específicos. Asimismo, se debe representar que un curso está compuesto por un horario definido.

El sistema será utilizado por un administrador académico y deberá diseñarse utilizando un enfoque orientado a objetos que permita identificar clases, atributos, métodos y relaciones como herencia, composición, asociación y dependencia entre los elementos del sistema.
